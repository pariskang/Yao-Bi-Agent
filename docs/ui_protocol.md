# UI Protocol

前端必须体现：规则透明、证据可追溯、医生可审核、患者不可直接照方使用。

## 页面建议

1. 医案输入页：自由文本、结构化表单、脱敏提示、信息完整度评分。
2. 结构化抽取页：展示“原文证据 → 标准标签”，允许医生修正。
3. 证型与方剂路线页：使用“候选证型”“经验规则匹配”“教学解释”，不用“确诊”。
4. 规则命中瀑布流：显示规则 ID、触发条件、命中证据、规则解释、风险提示。
5. 药物模块图谱页：显示模块、代表药物、触发证据、安全等级，不显示完整处方。
6. 复诊演变页：时间轴展示规则模块变化，不强调疗效因果。
7. 报告导出页：Markdown、PDF、DOCX、JSON、CSV。
